# Forge MVP 范围定义

> 日期: 2026-03-12
> 关联文档: `forge-boundary-mvp-roadmap.md`

## 一、文档目标

本文档用于明确 Forge v1 MVP 的必做范围、可延后范围和明确不做范围，避免在产品与架构推进过程中继续扩散。

## 二、MVP 定义原则

Forge MVP 只服务一个目标：

**让研发人员在单仓库上下文中，通过一个 AI 工作台完成一次完整交付任务。**

范围判断标准只有三条：

1. 是否直接服务“任务输入到结果确认”闭环
2. 是否会被核心用户高频使用
3. 是否是后续团队化和平台化演进的必要基础

## 三、目标用户

### 核心用户

- 研发工程师
- 技术负责人

### 非核心用户

- 高层管理者
- 跨部门流程审批人
- 纯业务侧需求方

## 四、核心任务类型

MVP 只支持以下任务类型：

1. 代码修改
2. 文档补充或更新
3. 配置调整
4. 缺陷修复
5. 小范围结构重构

暂不支持：

- 大规模项目拆解与排期管理
- 复杂跨仓库交付编排
- 长链路自动化发布流程
- 企业级经营分析

## 五、必做范围

### 5.1 任务闭环

- 用户输入任务目标
- 系统展示任务理解结果
- Agent 输出执行计划
- Agent 执行并修改工作区文件
- 系统展示过程日志与产物摘要
- 用户确认结果

### 5.2 工作台能力

- Web IDE 主界面
- 文件树浏览
- 文件内容查看
- 对话输入与会话展示
- 执行状态展示
- 结果确认区域

### 5.3 Agent 能力

- 单一 SuperAgent
- 基础计划生成
- 基础文件读写
- 基础命令执行
- 基础结果总结

### 5.4 上下文能力

- 当前仓库文件读取
- 项目规则文件读取
- 与任务相关文件选择
- 最近一次执行结果保留

### 5.5 基础工程连接

- 本地工作区访问
- Git 变更识别
- 基础 Shell 执行

## 六、可延后范围

以下内容有价值，但不进入 v1 MVP：

- 团队共享记忆
- 模板与基线中心化管理
- 轻量审批流
- 工具调用可视化增强
- 任务历史检索
- 多 profile 明确路由展示
- 更多外部系统连接

## 七、明确不做范围

以下内容不应在 v1 阶段投入：

- 多 Agent 协同网络
- 完整 MCP 平台矩阵
- 企业级权限与租户体系
- 项目管理看板
- 甘特图与经营分析
- 全自动发布编排
- 面向非研发岗位的通用门户

## 八、验收标准

满足以下条件可视为 MVP 成立：

1. 用户可在一个工作台中发起任务并获得可审阅结果
2. Agent 能稳定完成文件读取、修改、总结三个核心动作
3. 用户能明确知道 AI 做了什么、改了什么、下一步需要确认什么
4. 同一用户愿意在相似任务上重复使用

## 九、当前阶段建议

在后续设计与实现中，若新增需求无法直接支持本文件的“必做范围”，默认进入“可延后范围”，不直接纳入 v1。
